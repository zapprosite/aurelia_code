# ADR 20260327-Porteiro: Defesa Soberana contra Prompt Injection 🏮🛡️⚓

## Status
Proposto / Em Implementação

## Contexto
O ecossistema Aurélia lida com prompts complexos e indexação de centenas de skills externas. Para evitar ataques de *Prompt Injection* (ex: "ignore all previous instructions") sem depender de APIs de segurança proprietárias (Llama Guard Cloud, OpenAI Moderation), propomos uma camada de defesa 100% local.

## Decisão
Implementar um sistema de "Porteiro" (Gatekeeper) baseado em:
1.  **Qwen 2.5 0.5b**: Modelo ultra-leve rodando via Ollama para classificação binária ([SAFE]/[UNSAFE]).
2.  **Greeting Whitelist**: Implementação de bypass nativa em Go para saudações e comandos curtos triviais ("Oi", "Olá", "Hi"), reduzindo falsos positivos.
3.  **Redis Stack**: Cache de hashes SHA-256 para evitar re-análise de prompts idênticos, garantindo latência de milissegundos.
4.  **Refinamento de Prompt**: System prompt do sentinel ajustado para permitir interações básicas de conversação.
5.  **Integração Nativa**: Middleware posicionado antes de qualquer chamada ao LLM Core.

## Consequências
- **Positivas**: Soberania total de segurança, latência mínima via cache, custo zero de API.
- **Negativas**: Aumento marginal no consumo de RAM (400MB para o Qwen).

## Alternativas Consideradas
- **Regex/Filtros de Palavra**: Ineficazes contra ataques semânticos.
- **Llama Guard Cloud**: Quebra a premissa de Soberania (Tier 0).
