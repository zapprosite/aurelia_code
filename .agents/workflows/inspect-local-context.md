# Workflow: inspect-local-context

Passos:
1.  **Ler Autoridade**: Iniciar sempre por `AGENTS.md`.
2.  **Ler Regras**: Consultar `.agent/rules/` para diretrizes específicas.
3.  **Consultar PRD**: Entender a visão e os objetivos do projeto em `PRD.md`.
4.  **Explorar Estado**: Revisar `.context/docs/` e `.context/plans/` para entender o progresso atual.
5.  **Validar Dependências**: Verificar managers reais (npm, pip, etc.) antes de assumir pacotes.
