# 📐 Regra 06: Planejamento Antes do Código

Nenhum código significativo deve ser escrito sem um plano aprovado.

<directives>
1. **Atitude**: Crie sempre um `implementation_plan.md` em `.context/plans/` para tarefas complexas.
2. **Impacto**: Avalie efeitos colaterais em arquivos de configuração e infraestrutura.
3. **Aprovação**: Busque feedback do usuário ou valide contra o `PRD.md` antes de iniciar a "escrita pesada".
</directives>
